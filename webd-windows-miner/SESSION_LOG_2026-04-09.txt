Context
- Obiectiv: stabilizare trimitere tranzactii in webd-windows-miner (broadcast, status, UX) + pachetare executabil beta.
- Cerinte utilizator: progres clar in UI, checkpoint-uri salvate, build beta executabil.

Modificari implementate
1) Broadcast si detectie status tranzactie
- Socket-first broadcast catre noduri candidate, cu fallback pe HTTP/JSON-RPC.
- Detectie pending extinsa pe:
  - /transactions/pending/object
  - /transactions/pending
- Matching robust pentru txId in structuri serializate Buffer.

2) Corectie respingere "transaction is too old in pool"
- Cauza identificata: tranzactii semnate cu timeLock stale (0).
- Fix aplicat:
  - preluare inaltime lant din nod (/top, fallback /)
  - calcul timeLock live (top-1)
  - timeLock trimis explicit in builder la semnare.

3) UX status plata
- Etape afisate: Trimisa la nod -> In pending/mempool -> Confirmata in block.
- Progres polling vizibil in UI (incercare curenta / max).
- Stare explicita la finalul monitorizarii fara confirmare.

4) Confirmare inainte de trimitere
- Prompt DA/NU inainte de send:
  - afiseaza suma WEBD si adresa destinatar.
  - NU => transfer anulat.

5) Fereastra de verificare extinsa
- Polling tranzactie extins la 60 verificari (~10 minute la 10s).

6) Operare/dev runtime
- Procese stale Node/Electron identificate si oprite.
- Restart curat din proiectul corect webd-windows-miner.
