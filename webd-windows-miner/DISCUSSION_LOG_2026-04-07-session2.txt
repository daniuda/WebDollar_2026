Project: webd-windows-miner

User issue identified
- Wallet showing incoming WEBD but not updating in application display without manual refresh.
- Need to verify if mining uses the new balance.

Investigation findings
- UI refresh logic: wallet values fetched from pool endpoints at startup, during mining loop refresh, and on manual `Refresh pool`.
- Mining logic: PoS uses balance from job/work payload delivered by pool; balance is updated when new work arrives.
- Problem: no continuous heartbeat refresh for wallet reward values when user idle/not mining.

Solution implemented (Checkpoint 036)
- Added periodic wallet reward refresh every 20 seconds via timer.
- Added immediate refresh triggers on pool URL or wallet address config changes.
- Result: incoming WEBD now visible in UI with max 20-second delay, independent of mining activity.
- Mining continues to use fresh balance from job payloads, unaffected by this UI change.

Installer issue reported by user
- Symptom: after installation, app opens with a black screen.

Diagnosis
- Packaged renderer was trying to load JS/CSS from absolute paths (`/assets/...`).
- In installed Electron app (`file://`), absolute `/assets/...` does not resolve to bundled files.

Fix applied
- Updated `vite.config.ts` with `base: './'`.
- Rebuilt project and confirmed generated `dist/index.html` now uses relative paths (`./assets/...`).

Verification
- Full build passed after fix (`npm run build`).

Status
- Black-screen root cause addressed in source; next installer build should include this fix.
