Flux propus - produs principal: pagina de minat, cu pool registry ca modul secundar
Data: 2026-04-04

Principiu
- Pagina principala nu este un registry general.
- Pagina principala este despre minat: onboarding, alegere pool, wallet, auto-start mining, stats.
- Registry-ul de pool-uri este o functie secundara care ajuta utilizatorul sa aleaga un pool verificat.

Obiectiv UX
- Utilizatorul ajunge rapid la actiunea principala: sa inceapa minatul.
- Daca wallet-ul este deja importat si utilizatorul indeplineste conditiile PoS, minatul trebuie sa porneasca automat la deschiderea paginii.
- Pool-urile propuse/verificate sunt integrate in fluxul de alegere a pool-ului, nu devin centrul produsului.

Regula de auto-start
- Nu exista buton principal de `Start Mining` in fluxul ideal.
- Daca exista wallet importat/deblocat si soldul eligibil este cel putin `100 WEBD`, pagina trebuie sa porneasca automat minatul.
- Pool-ul se alege aleator din lista de pool-uri `verified` sau `recommended` care sunt online si trec pragul minim de health.
- Daca nu exista wallet importat sau soldul este sub prag, pagina intra in mod ghidat de onboarding/setup.

Originea pragului de 100 WEBD
- Pragul de `100 WEBD` nu este o regula inventata de UI.
- In codul WebDollar de baza, regula PoS vine din `Node-WebDollar/src/consts/const_global.js`, unde `POS.MINIMUM_AMOUNT = 100`.
- Verificarea efectiva apare in `Node-WebDollar/src/common/blockchain/interface-blockchain/mining/Interface-Blockchain-Mining.js`, unde mineritul PoS este refuzat sub acest prag si este afisat mesajul `It is required 100 WEBD to mine POS`.
- Concluzie: pragul de 100 WEBD este cerut de logica de protocol/core mining PoS, iar pool-ul sau UI-ul doar trebuie sa respecte aceasta regula.

Structura principala de pagini
1. Home / Start Mining
- mesaj clar: incepe sa minezi WebDollar
- alegere rapida wallet + pool + download miner / launch guide
- top stats sumar: pool-uri online, payout info, mineri activi, uptime
- daca wallet-ul este eligibil, pagina intra direct in starea `Mining now`
- CTA secundar: `Vezi pool-uri verificate`

2. Choose Pool
- lista de pool-uri recomandate/verificate
- carduri cu status, fee, payout, uptime, nr. mineri, latenta
- filtru simplu: recommended / all / low fee / low latency
- actiune pe fiecare card: `Mineaza cu acest pool`

3. Pool Details
- informatii detaliate pentru un pool selectat
- endpoint-uri publice
- health status
- istoric uptime
- payout info
- buton principal: `Foloseste acest pool pentru minat`

4. Submit Pool
- pagina secundara pentru propuneri noi
- formular public
- mesaj clar ca listarea se face doar dupa verificare automata

5. Mining Help / Setup
- instructiuni de setup
- wallet help
- explicatii despre payout, fee, status pool

Fluxul principal pentru utilizator
Flux A - utilizator nou
1. Intra pe Home.
2. Alege sau importa wallet.
3. Daca soldul PoS eligibil este sub `100 WEBD`, primeste explicatie si ghid de setup.
4. Daca soldul este eligibil, sistemul alege automat un pool verificat din lista activa.
5. Minatul porneste automat.
6. Utilizatorul vede stats si poate schimba manual pool-ul doar optional.

Flux B - utilizator care vrea doar un pool bun
1. Intra pe Home.
2. Apasa `Vezi pool-uri verificate`.
3. Filtreaza pool-urile.
4. Deschide `Pool Details`.
5. Alege manual un pool alternativ fata de cel ales automat.

Flux C - operator de pool
1. Intra pe `Submit Pool`.
2. Completeaza formularul.
3. Pool-ul intra in `pending`.
4. Sistemul face verificare automata.
5. Daca trece, apare in `Choose Pool` si in lista publica.

Ce trebuie sa se vada pe Home
- headline despre minat
- explicatie scurta despre auto-start daca wallet-ul este eligibil
- bloc simplu cu `1. Wallet 2. Eligibilitate 3. Mining automat`
- shortlist cu 3-6 pool-uri recomandate
- statut general retea/pool-uri
- link spre documentatie si download miner

Ce trebuie sa NU domine Home
- tabele administrative lungi
- formularul de submit pool direct in prim-plan
- incidente tehnice detaliate
- statusuri interne prea tehnice fara context

Informatii utile pe cardul de pool
- nume pool
- badge status: verified / degraded / offline
- fee
- payout minim
- uptime recent
- latenta estimata
- nr. mineri sau capacitate publica daca exista
- optiune `Selecteaza manual` doar ca override

Statusuri potrivite pentru UI mining-first
- Recommended
- Verified
- Degraded
- Offline
- New

Reguli de prioritate pentru listare
1. Pool-uri `Recommended` sus
2. Pool-uri `Verified` dupa scor
3. Pool-uri `New` mai jos, daca au trecut pragul minim
4. Pool-uri `Degraded` separat
5. Pool-uri `Offline` ascunse implicit sau intr-o sectiune separata

Reguli pentru alegerea automata a pool-ului
1. Se folosesc doar pool-uri `Recommended` sau `Verified`.
2. Pool-ul trebuie sa fie `online` si peste pragul minim de health.
3. Alegerea este aleatoare din setul eligibil, nu hardcodată.
4. Optional, randomizarea poate fi ponderata ulterior dupa health score, fee si latenta.
5. Daca nu exista pool eligibil, pagina afiseaza fallback de selectie manuala.

Componente frontend recomandate
- MiningHero
- QuickStartSteps
- RecommendedPoolsStrip
- PoolCard
- PoolStatusBadge
- PoolFilters
- PoolDetailsPanel
- SubmitPoolForm
- HelpAccordion

Legatura cu backend-ul
- frontendul cere doar datele necesare pentru UX de minat
- backendul decide cine este verified/recommended
- frontendul nu decide singur increderea unui pool

Recomandare de produs
- Daca facem site-ul, numele si navigatia trebuie sa porneasca de la `Start Mining`, nu de la `Pool Registry`.
- Registry-ul trebuie sa existe, dar ca infrastructura de suport pentru alegerea unui pool bun.

Concluzie
- Produsul principal: pagina de minat.
- Modul secundar: registry de pool-uri verificate.
- UX-ul trebuie sa impinga utilizatorul spre wallet + eligibilitate + auto-mining, nu spre browsing administrativ.
